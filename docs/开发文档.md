# 微信公众号助手开发文档

## 1. 项目定位

`wechat-official-account-skill` 是一个面向 AI CLI/Agent 的微信公众号运营技能项目。它把公众号内容生产过程中的模板写作、素材生成、素材上传、公众号排版、草稿创建、发布、菜单、用户和数据统计统一封装为本地 CLI。

核心目标：

1. 让 AI 能通过稳定命令完成公众号运营任务。
2. 让文章模板、素材库、图片生成和微信 API 形成闭环。
3. 降低人工复制粘贴、找图、排版和创建草稿的重复成本。
4. 保持所有敏感配置在本地 `.env` 中，不在日志和文档中输出密钥。

## 2. 技术栈

| 类型 | 技术 |
|---|---|
| 语言 | Python 3 |
| CLI 框架 | Click |
| HTTP 请求 | Requests |
| 环境变量 | python-dotenv |
| Markdown 转 HTML | Mistune |
| HTML 处理 | BeautifulSoup + html5lib |
| 配置/索引 | YAML |
| 微信 API | 公众号素材、草稿、菜单、用户、数据统计接口 |
| 图片生成 | OpenAI 兼容 `/images/generations` 接口 |

## 4. 配置系统

配置读取入口在 `src/auth.py`：

```python
load_config()
```

它会读取项目根目录下的 `.env`。

### 4.1 微信配置

```env
WECHAT_APP_ID=公众号 AppID
WECHAT_APP_SECRET=公众号 AppSecret
WECHAT_TOKEN=服务器配置 Token，可选
WECHAT_AES_KEY=服务器配置 EncodingAESKey，可选
THEME=default
FOOTER_SIGNATURE=true
```

### 4.2 图片生成配置

```env
IMAGE_API_BASE_URL=https://api.openai.com/v1
IMAGE_API_ENDPOINT=/images/generations
IMAGE_API_KEY=OpenAI 或兼容服务 API Key
IMAGE_MODEL=gpt-image-1
IMAGE_SIZE=1024x1024
IMAGE_RESPONSE_FORMAT=
IMAGE_API_EXTRA_JSON=
```

说明：

- `IMAGE_API_BASE_URL` 支持 OpenAI 兼容服务地址。
- `IMAGE_API_ENDPOINT` 默认 `/images/generations`。
- `IMAGE_RESPONSE_FORMAT` 可按兼容接口要求设置为 `url` 或 `b64_json`，留空则由接口默认决定。
- `IMAGE_API_EXTRA_JSON` 可传附加 JSON，例如：

```env
IMAGE_API_EXTRA_JSON={"quality":"high"}
```

## 5. access_token 缓存

`src/auth.py` 使用 `.token_cache.json` 缓存微信 `access_token`。

主要函数：

| 函数 | 作用 |
|---|---|
| `read_cached_token()` | 读取未过期 token |
| `write_cached_token(access_token, expires_in)` | 写入 token 和过期时间 |
| `require_credentials()` | 检查微信凭证是否存在 |

缓存策略：

- 提前 60 秒视为过期。
- 写入时默认减少 120 秒缓冲，避免临界过期。

## 6. 微信 API 客户端

`src/client.py` 中的 `WechatClient` 封装所有微信 API 请求。

### 6.1 核心方法

| 方法 | 作用 |
|---|---|
| `get_access_token()` | 获取或刷新 token |
| `get(path, params=None)` | 调用 GET API |
| `post(path, payload=None, params=None)` | 调用 POST API |
| `post_file(path, file_path, field_name='media', params=None)` | 上传文件 |

### 6.2 中文编码修复

微信草稿接口如果使用 `requests.post(json=...)`，中文可能被序列化为 `\uXXXX` 并在草稿中原样显示。

当前实现使用：

```python
body = json.dumps(payload or {}, ensure_ascii=False).encode("utf-8")
requests.post(..., data=body, headers={"Content-Type": "application/json; charset=utf-8"})
```

这可以确保公众号草稿中正常显示中文。

## 7. 模板系统

模板位于 `templates/`，每个模板是 Markdown 文件，并使用 YAML frontmatter 描述元信息。

示例：

```markdown
---
name: 技术文章
category: 技术分享
theme: modern
style: tech
materials: [banner, icon, hero, divider]
variables:
  - name: title
    desc: 文章标题
    default: "一次技术实践分享"
---

![技术横幅]({{ material_banner }})

# {{ title }}
```

### 7.1 模板字段

| 字段 | 说明 |
|---|---|
| `name` | 展示名称，可用于 `template show` |
| `category` | 分类，如技术分享、产品运营 |
| `theme` | 默认排版主题 |
| `style` | 风格标签，用于素材匹配 |
| `materials` | 需要注入的素材槽位 |
| `variables` | 可替换变量列表 |

### 7.2 变量替换

变量格式：

```text
{{ variable_name }}
```

渲染时传入：

```bash
python bin/wechat template render tech_article \
  --var title="标题" \
  --var summary="摘要"
```

未传入变量时使用模板中的 `default`。

### 7.3 自动素材变量

模板声明：

```yaml
materials: [banner, icon, hero, divider, cover]
```

渲染时会自动注入：

```text
{{ material_banner }}
{{ material_icon }}
{{ material_hero }}
{{ material_divider }}
{{ material_cover }}
```

## 8. 本地素材库

素材库位于：

```text
assets/library/
```

索引文件：

```text
assets/library/index.yaml
```

索引结构：

```yaml
assets:
  - id: tech_banner
    slot: banner
    path: banners/tech_banner.png
    tags: [技术分享, tech_article, modern, tech]
```

### 8.1 素材槽位

| 槽位 | 用途 |
|---|---|
| `cover` | 封面图 |
| `banner` | 正文顶部横幅 |
| `hero` | 主题插图 |
| `icon` | 图标 |
| `divider` | 分隔图 |
| `background` | 背景素材 |

### 8.2 自动选择逻辑

实现位置：`src/material_library.py`

匹配依据：

1. 素材 `slot` 必须与模板需要的槽位一致。
2. 根据模板 `id/category/theme/style` 与素材 `tags` 评分。
3. 如果多个素材同分，使用模板信息做稳定 hash 选择。
4. 如果没有匹配素材，使用默认素材。

核心函数：

| 函数 | 作用 |
|---|---|
| `load_asset_index()` | 读取素材索引 |
| `select_asset(slot, template_meta)` | 选择单个素材 |
| `select_materials(template_meta)` | 为模板选择全部素材 |

## 10. Markdown 转公众号 HTML

入口：

```python
markdown_to_wechat_html(markdown_text, theme_name=None, footer=None)
```

功能：

1. 使用 Mistune 将 Markdown 转 HTML。
2. 使用 BeautifulSoup 添加内联样式。
3. 根据主题处理标题、段落、引用、代码、图片等样式。
4. 添加页脚签名。
5. 将列表转换成段落模拟列表，避免公众号编辑器出现隔行空白。

### 10.1 列表兼容策略

公众号编辑器对 `<ul>/<ol>/<li>` 的兼容性可能导致单数行空白。

当前实现会将列表转成：

```html
<p style="margin: 0 0 8px; padding-left: 1.2em; text-indent: -1.2em">• 列表项</p>
```

有序列表转成：

```html
<p>1. 列表项</p>
```

## 11. 图片生成系统

实现位置：`src/image_generation.py`

支持 OpenAI 兼容 `/images/generations` 接口。

### 11.1 图片生成

```bash
python bin/wechat image generate --prompt "提示词"
```

只保存本地图片。

### 11.2 封面生成并上传

```bash
python bin/wechat image cover --prompt "封面提示词"
```

流程：

1. 调用图片生成 API。
2. 保存到本地。
3. 调用微信素材上传接口。
4. 返回 `media_id` 和本地路径。

### 11.3 正文插图生成

```bash
python bin/wechat image illustrate --file article.md --max-images 4 --upload
```

流程：

1. 读取 Markdown。
2. 按 `#` / `##` / `###` 标题提取段落。
3. 为每个段落生成图片提示词。
4. 调用图片生成 API。
5. 保存本地图片。
6. 如果加 `--upload`，上传到微信素材库。
7. 生成 `*_illustrated.md`。

## 12. 草稿系统

实现位置：`src/draft.py`

| 函数 | 作用 |
|---|---|
| `create_draft()` | 创建草稿 |
| `list_drafts()` | 查询草稿 |
| `delete_draft()` | 删除草稿 |
| `publish_draft()` | 发布草稿 |

### 12.1 创建草稿

```bash
python bin/wechat draft create \
  --title "文章标题" \
  --content-file article.md \
  --cover-media-id MEDIA_ID \
  --author "AI助手" \
  --digest "摘要" \
  --no-comment
```

如果 `content-file` 是 Markdown，会自动转换成公众号 HTML。

如果是 `.html` / `.htm`，直接作为 HTML 提交。

### 12.2 删除草稿

```bash
python bin/wechat draft delete <draft_media_id>
```

此操作不可恢复。

### 12.3 发布草稿

```bash
python bin/wechat draft publish <draft_media_id>
```

发布属于外部可见操作，执行前应确认用户明确授权。

## 15. 新增模板步骤

1. 在 `templates/` 新建 `.md` 文件。
2. 添加 YAML frontmatter。
3. 声明 `name/category/theme/style/materials/variables`。
4. 在正文中使用变量和素材变量。
5. 运行验证：

```bash
python bin/wechat template show 模板ID
python bin/wechat template render 模板ID --output articles/test.md
```

## 16. 新增素材步骤

1. 将图片放入 `assets/library/<slot>/`。
2. 编辑 `assets/library/index.yaml`。
3. 添加素材记录：

```yaml
- id: new_asset
  slot: banner
  path: banners/new_asset.png
  tags: [技术分享, modern, tech]
```

4. 渲染模板验证是否被选中。

## 17. 新增主题步骤

1. 在 `themes/` 新建 `theme_name.yaml`。
2. 按标签写样式。
3. 在模板 frontmatter 中设置：

```yaml
theme: theme_name
```

4. 转换验证：

```bash
python bin/wechat convert --file article.md --theme theme_name > article.html
```

## 19. 安全与注意事项

1. 不要输出 `.env` 中的 `WECHAT_APP_SECRET` 或 `IMAGE_API_KEY`。
2. 不要提交 `.env`、`.token_cache.json`。
3. 创建草稿是可见于公众号后台的操作，但不会发布。
4. 发布、删除全部草稿、批量操作都属于高风险动作，必须确认。
5. 图片生成会消耗 API 额度。
6. 上传图片会新增微信素材。
7. 删除草稿不可恢复。

# 微信公众号助手使用文档

## 1. 工具简介

微信公众号助手是一个本地命令行工具，用于辅助完成公众号文章写作、自动配图、排版、素材上传、草稿创建、发布、菜单、用户和数据统计。

你可以把它理解成一条公众号内容生产流水线：

```text
选择模板 → 写文章 → 自动选择素材 → 生成插图/封面 → 上传素材 → 创建公众号草稿 → 人工检查 → 发布
```

统一入口：

```bash
python bin/wechat --help
```

## 2. 初始化配置

进入项目目录（根据你保存技能的位置）：

```bash
cd 你的技能目录/wechat-official-account-skill
```

> **注意**：项目目录取决于你保存技能的位置，请根据实际情况调整路径。

安装依赖：

```bash
pip install -r requirements.txt
```

复制配置文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入公众号和图片生成 API 配置。

## 3. 配置说明

### 3.1 微信公众号配置

```env
WECHAT_APP_ID=你的公众号 AppID
WECHAT_APP_SECRET=你的公众号 AppSecret
WECHAT_TOKEN=your_token_here
WECHAT_AES_KEY=your_aes_key_here
THEME=default
FOOTER_SIGNATURE=true
```

必填：

```text
WECHAT_APP_ID
WECHAT_APP_SECRET
```

可选：

```text
WECHAT_TOKEN
WECHAT_AES_KEY
THEME
FOOTER_SIGNATURE
```

### 3.2 图片生成 API 配置

```env
IMAGE_API_BASE_URL=https://api.openai.com/v1
IMAGE_API_ENDPOINT=/images/generations
IMAGE_API_KEY=你的图片生成 API Key
IMAGE_MODEL=gpt-image-1
IMAGE_SIZE=1024x1024
IMAGE_RESPONSE_FORMAT=
IMAGE_API_EXTRA_JSON=
```

如果你使用第三方兼容 OpenAI 的图片接口，只需要修改：

```env
IMAGE_API_BASE_URL=https://你的服务地址/v1
IMAGE_API_KEY=你的 key
IMAGE_MODEL=对应模型名
```

## 6. 使用模板生成文章

基础命令：

```bash
python bin/wechat template render 模板ID --output 输出文件.md
```

示例：

```bash
python bin/wechat template render tech_article \
  --var title="从模板到素材：公众号自动排版实践" \
  --var summary="复盘一次自动化公众号内容工作流。" \
  --var problem="模板过于简约，素材需要人工插入。" \
  --var solution="通过模板声明素材槽位，自动匹配本地素材库。" \
  --var result="文章生成后自动带有横幅、插图和分隔图。" \
  --output articles/my_tech_article.md
```

## 7. 自动素材怎么使用

模板中会声明需要的素材：

```yaml
materials: [banner, icon, hero, divider]
```

渲染时系统会自动从本地素材库选择图片，并替换为路径：

```markdown
![技术横幅](assets/library/banners/tech_banner.png)
```

本地素材库目录：

```text
assets/library/
```

常见素材类型：

| 类型 | 用途 |
|---|---|
| `covers` | 封面图 |
| `banners` | 正文横幅 |
| `illustrations` | 正文主题插图 |
| `icons` | 图标 |
| `dividers` | 分隔图 |
| `backgrounds` | 背景素材 |

## 8. 生成本地图片

只生成图片，不上传微信：

```bash
python bin/wechat image generate \
  --prompt "极简科技图标，代码括号与芯片融合，蓝紫渐变，无文字无水印" \
  --output-dir assets/library/generated \
  --filename tech_icon_new.png
```

参数说明：

| 参数 | 说明 |
|---|---|
| `--prompt` | 图片提示词 |
| `--output-dir` | 保存目录 |
| `--filename` | 文件名 |
| `--model` | 临时覆盖模型 |
| `--size` | 临时覆盖尺寸 |
| `--n` | 生成数量 |
| `--response-format` | 返回格式 |

## 9. 生成并上传封面

```bash
python bin/wechat image cover \
  --prompt "公众号封面图，AI 编程与软件开发，蓝紫科技风，16:9，无文字无水印"
```

成功后返回：

```json
[
  {
    "media_id": "...",
    "url": "...",
    "local_path": "..."
  }
]
```

其中 `media_id` 可用于创建草稿：

```bash
--cover-media-id MEDIA_ID
```

## 10. 根据文章自动生成正文插图

命令：

```bash
python bin/wechat image illustrate \
  --file articles/my_article.md \
  --max-images 4 \
  --style "现代科技感扁平插画，16:9，公众号正文配图，蓝紫渐变，无文字无水印" \
  --upload
```

作用：

1. 读取 Markdown 文章。
2. 根据标题和段落内容生成插图提示词。
3. 调用图片生成 API。
4. 保存本地图片。
5. 上传微信素材库。
6. 生成带插图的新 Markdown 文件。

输出示例：

```text
articles/my_article_illustrated.md
articles/my_article_images/
```

如果不加 `--upload`，图片只保存在本地，Markdown 中插入本地路径。

## 11. Markdown 转公众号 HTML

```bash
python bin/wechat convert --file articles/my_article.md --theme modern > articles/my_article.html
```

不添加页脚：

```bash
python bin/wechat convert --file articles/my_article.md --theme modern --no-footer > articles/my_article.html
```

主题可选：

```text
default
modern
minimal
neon
clean
warm
```

## 12. 上传本地图片到微信素材库

```bash
python bin/wechat upload image --file assets/library/covers/tech_cover.png
```

返回结果中重点看：

```text
media_id
url
```

- `media_id`：可用于封面。
- `url`：可用于正文图片引用。

## 13. 创建公众号草稿

创建草稿前需要有封面 `media_id`。

```bash
python bin/wechat draft create \
  --title "文章标题" \
  --content-file articles/my_article.md \
  --cover-media-id MEDIA_ID \
  --author "AI助手" \
  --digest "文章摘要" \
  --no-comment
```

说明：

- 如果 `content-file` 是 Markdown，会自动转换为公众号 HTML。
- 如果 `content-file` 是 HTML，会直接提交 HTML。
- 创建草稿不会发布文章。

## 22. 常见问题

### 22.1 草稿里中文显示成 `\u5f53\u66f4\u5f3a`

原因：JSON 提交时中文被 ASCII 转义。

当前版本已修复，使用 UTF-8 JSON 提交。

### 22.2 列表出现隔行空白

原因：公众号编辑器对 `<ul>/<ol>/<li>` 兼容不稳定。

当前版本会把列表转换为段落模拟列表，避免空行问题。

### 22.3 图片没有显示

可能原因：

1. Markdown 中引用的是本地路径，公众号后台无法访问。
2. 图片没有上传到微信素材库。
3. 图片 URL 失效或不是微信素材 URL。

解决：

```bash
python bin/wechat upload image --file 图片路径
```

然后把正文中的图片链接替换为返回的 `url`。

### 22.4 图片生成失败

检查：

1. `.env` 中 `IMAGE_API_KEY` 是否正确。
2. `IMAGE_API_BASE_URL` 是否是兼容 OpenAI 的地址。
3. `IMAGE_MODEL` 是否存在。
4. 当前服务是否支持 `/images/generations`。
5. 额度是否充足。

### 22.5 创建草稿失败：标题或摘要过长

微信对标题和摘要长度有限制。建议：

- 标题尽量控制在 30 字以内。
- 摘要尽量短，不要写成长段。

## 23. 安全提醒

1. 不要把 `.env` 发给别人。
2. 不要在聊天里粘贴 `WECHAT_APP_SECRET` 和 `IMAGE_API_KEY`。
3. 发布文章前必须人工检查内容。
4. 删除草稿、发布草稿、修改菜单属于高风险操作。
5. 图片生成和微信素材上传都会消耗资源或产生后台记录。
